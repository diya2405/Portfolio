# Diya Shah – Software Developer Portfolio

A modern, production-ready personal portfolio website built with React + Vite, showcasing skills, projects, and experience.

## Tech Stack

- **React 19** + **Vite 7** – Fast frontend tooling
- **Tailwind CSS 3** – Utility-first styling with dark mode
- **Framer Motion** – Smooth animations and transitions
- **react-type-animation** – Typing effect in the hero section
- **react-icons** – Icon library

## Features

- 🌙 Dark/Light mode toggle
- 🎨 Glassmorphism + gradient design
- 📱 Fully responsive (mobile-first)
- ⚡ Animated sections on scroll (Framer Motion)
- 🖥️ Hero with typing animation and floating background orbs
- 🗂️ Projects section with animated cards and detail modals
- 📊 Animated skill progress bars
- 📈 GitHub stats integration
- 📬 Contact form with mailto integration
- 🔝 Back-to-top button + loading screen
- 🔍 SEO meta tags

## Sections

1. **Hero** – Name, animated role titles, CTA buttons, profile image
2. **About** – Bio, skill highlights, education timeline
3. **Skills** – Animated progress bars across Languages, Frameworks, AI/Backend
4. **Projects** – JobFlow, CodeQuest, AI Assistant App, Sentiment Analysis
5. **Experience** – Internship, certifications, academic GPA
6. **GitHub Stats** – Embedded contribution stats
7. **Contact** – Form + social links
8. **Footer** – Minimal with social icons

## Getting Started

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

## Deployment

The `dist/` folder (generated by `npm run build`) can be deployed to any static hosting provider (Netlify, Vercel, GitHub Pages, etc.).

### Deploy to GitHub Pages (from GitHub)

1. Push to the `main` branch (or run the workflow manually from the Actions tab).
2. In GitHub **Settings → Pages**, set the source to **GitHub Actions**.
3. The workflow in `.github/workflows/deploy.yml` will build and publish the `dist/` folder.

The workflow sets `BASE_PATH` to `/<repo-name>/` so assets resolve correctly on project pages. If you deploy to a custom domain or a user site (`username.github.io`), set `BASE_PATH=/` in the workflow (or remove it) to serve from the root.
